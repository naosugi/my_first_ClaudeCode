## このガイドで伝えたいこと

**コマンドを覚える必要はありません。**

Claude Codeは「自然言語で話しかければ動く」ツールです。
このガイドでは、コマンドの暗記ではなく、**考え方**を身につけることを目指します。

最終的なゴール：
> **「自然な欲求を、いちいち説明しなくても実現できる状態」**

## なぜClaude Codeなのか

### ChatGPTやClaude.aiとの違い

| | ChatGPT / Claude.ai | Claude Code |
|---|---------------------|-------------|
| 場所 | ブラウザ | ターミナル（黒い画面） |
| できること | 会話、テキスト生成 | ファイル操作、コマンド実行、プロジェクト全体の理解 |
| 本質 | **アドバイザー** | **実行者** |

ChatGPTは「こうすればいいですよ」と教えてくれます。
Claude Codeは「やっておきました」と実際に手を動かします。

### 専門家チームを召喚できる

Claude Codeの真価は**並列実行**にあります。

```
「5人の専門家視点で、このビジネスプランを評価して：
 - 財務の専門家
 - マーケティングの専門家
 - 技術の専門家
 - 法務の専門家
 - ユーザー体験の専門家」
```

これだけで、5つの視点からの分析が並列で実行されます。
専門家を雇うのではなく、**召喚する**感覚です。

## 2つの原則

### 原則1: AIも「うっかり」する → 仕組みで防ぐ

AIは万能ではありません。長い会話の途中で、最初の指示を忘れることがあります。

**対策**：
- 重要なルールは `CLAUDE.md` に書いておく（AIの「憲法」）
- 複雑な作業は「並列サブエージェント」に分割する

これは「気をつけて」ではなく「仕組み」で解決する考え方です。

### 原則2: 覚えるより、育てる

コマンドを暗記するのではなく、**自分の欲求を育てていく**。

```
Week 1: 「競合を調べて」と毎回言う
Week 2: 「毎回同じこと言うの面倒だな」
Week 3: この欲求を保存 → 自分専用のショートカットになる
```

後述する「カスタムコマンド」は、この**欲求を保存する仕組み**です。

## セットアップ

### 必要なもの

| 必要なもの | 確認方法 |
|-----------|----------|
| Mac | 今使っているPCがMacならOK |
| Anthropicアカウント | [claude.ai](https://claude.ai) で登録 |
| 有料プラン | Pro($20/月) または Max($100/月) |
| Node.js | ターミナルで `node -v`（v18以上） |

### インストール（3ステップ）

**ステップ1**: ターミナルを開く
- `Command + Space` → 「ターミナル」と入力 → Enter

**ステップ2**: Claude Codeをインストール
```bash
npm install -g @anthropic-ai/claude-code
```

**ステップ3**: 起動して認証
```bash
claude
```
ブラウザが開くので、Anthropicアカウントでログイン。

## 実践：自然言語で指示する

### 基本：話しかけるだけ

```bash
cd ~/Documents/my-project  # 作業したいフォルダに移動
claude                      # Claude Codeを起動
```

あとは話しかけるだけ：

```
このフォルダの構成を説明して
```

```
READMEファイルを作って
```

```
このコードのバグを直して
```

### 応用：専門家チームを召喚

**複数視点での分析**：
```
3人の専門家視点で、このアイデアを評価して：
- 技術的な実現可能性
- ビジネスとしての収益性
- ユーザーにとっての価値
```

**並列リサーチ**：
```
以下の会社を並列で調査して：
- 会社A: 製品、最近の動向、強み
- 会社B: 同上
- 会社C: 同上
最後に比較表を作って。
```

**並列タスク実行**：
```
4つのタスクを並列で実行して：
1. テストを実行
2. ドキュメントを更新
3. コードをフォーマット
4. 依存関係をチェック
```

### 覚えておくと便利な操作

| やりたいこと | 方法 |
|-------------|------|
| 終了する | `Ctrl + C` または `/exit` |
| 前回の続きから | `claude --resume` |
| コスト確認 | `/cost` |

## 成長：欲求を保存する

### 「毎回言うの面倒」が成長のサイン

最初は自然言語で指示します。
そのうち「毎回同じこと言ってるな」と気づきます。

**それが、欲求を保存するタイミングです。**

### 欲求の保存方法

例：「競合調査」を毎回頼んでいる場合

**Step 1**: いつも言っている内容を思い出す
```
「〇〇社を調査して、製品・資金調達・顧客評価を分析して」
```

**Step 2**: ファイルに保存
```bash
mkdir -p .claude/commands
```

`.claude/commands/research-competitor.md` を作成：
```markdown
# 競合調査

以下の会社を調査して：$ARGUMENTS

各社について分析：
- 主要製品とサービス
- 最近の資金調達
- 顧客からの評価

最後に比較表を作成。
```

**Step 3**: 使う
```
/project:research-competitor A社, B社, C社
```

### これがカスタムコマンド

「カスタムコマンドを作る」と構えなくて大丈夫です。

**順序が逆**なのです：
1. ❌ 最初にコマンドを設計する
2. ⭕ 自然言語で使ってみて、パターンが見えたら保存する

**欲求が先、コマンドは後。**

## 補足：CLAUDE.mdとは

プロジェクトのルートに `CLAUDE.md` を置くと、AIが毎回それを読みます。

**用途**：
- 「このプロジェクトではTypeScriptを使う」
- 「テストは必ず書く」
- 「この形式でコミットメッセージを書く」

自動生成するには：
```
/init
```

詳細は [セットアップガイド](./setup-guide.md) を参照。
