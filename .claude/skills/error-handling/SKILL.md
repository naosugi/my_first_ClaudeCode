# エラーハンドリング標準パターン

## 1. エラー発生時の初動

1. **エラーメッセージを正確に記録**
   - 完全なエラーメッセージをコピー
   - 発生箇所（ファイル:行番号）を特定
   - 再現手順を記録

2. **影響範囲の確認**
   - 他の機能への影響はあるか
   - データ損失の可能性はあるか
   - セキュリティリスクはあるか

## 2. 調査手順

1. **公式ドキュメントを確認**
   - claude-code-guide エージェントに仕様確認
   - エラーメッセージで検索

2. **類似事例を確認**
   - troubleshooting.md を検索
   - progress.md の過去ログを確認

3. **根本原因を特定**
   - 表面的な対処ではなく、根本原因を追求
   - 「なぜ」を5回繰り返す

## 3. 修正と永続化

1. **修正を実施**
   - 最小限の変更で解決
   - 副作用がないか確認

2. **学習を永続化**
   - troubleshooting.md に事例追加
   - 3回以上発生 → スキル化候補

## 4. 報告フォーマット

```
## エラー報告

- **発生日時**: YYYY-MM-DD HH:MM
- **エラー内容**: {メッセージ}
- **発生箇所**: {ファイル:行番号}
- **根本原因**: {原因}
- **対処**: {実施した修正}
- **永続化**: {troubleshooting.md / スキル化 / なし}
```
